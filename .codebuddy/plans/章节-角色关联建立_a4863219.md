## 产品概述

为「没钱修什么仙-番剧制作」项目的章节摘要补充角色关联链接。当前章节摘要的「关联」区域缺少角色文件链接（或链接不完整），需要将出现在章节中的角色与 `20-角色/` 中的角色档案文件建立 Wiki 链接关联。

## 核心功能

1. **角色识别**：从章节摘要的「出场角色」板块提取所有角色引用（Wiki链接或文件路径）
2. **角色链接补充**：将缺失的角色Wiki链接追加到「关联」区域末尾，保留所有现有非角色链接
3. **遗漏角色新建**：对原文中出现但角色目录中不存在的新角色，创建标准角色档案文件
4. **增量幂等处理**：重复运行不产生重复链接，不删除任何已有内容

## 技术栈

- **语言**：Python 3（项目 `90-进度追踪/` 中已有 .py 文件，复用现有环境）
- **依赖**：仅使用 Python 标准库 os, re, pathlib（无需安装第三方包）
- **运行环境**：Windows PowerShell

## 核心架构

### 整体流程

```
[角色档案目录] --> [建立角色名称索引] --> [逐章处理]
                                            |
                                     [读取章节摘要]
                                            |
                              [解析"出场角色"板块 -> 提取角色路径]
                                            |
                              [解析"关联"板块 -> 提取已有链接]
                                            |
                              [对比 -> 补充缺失的角色链接]
                                            |
                                     [写入更新后的文件]
```

### 关键数据结构

**角色名称索引**：`Dict[str, str]` — 将角色可识别名称映射到文件路径

- 示例：`{"张羽": "20-角色/主角/张羽", "白真真": "20-角色/主角/白真真", "母亲": "20-角色/配角/人物-41-母亲"}`

**已有链接集合**：`Set[str]` — 从「关联」板块提取的现有路径集合
**待补充链接**：`Set[str]` — 角色索引中存在但不在已有链接集合中的路径

### 文件处理策略

对 `30-剧情/章节摘要/` 下每个文件（~200个）：

1. 定位「出场角色」板块：匹配 `## 出场角色` 到下一个 `##` 之间的内容
2. 解析角色引用：支持三种格式：

- `[[20-角色/主角/张羽]]` — 完整 WikiLink
- `20-角色/配角/人物-22-赵天行` — 裸文件路径
- `车女（暗学帮走廊说明）` — 纯文本名称（需用角色名称索引匹配）

3. 定位「关联」板块：从 `## 🔗 关联`（或 `## 关联`）到文件末尾
4. 补充缺失链接：所有缺失的角色路径以 `- [[path]]` 格式追加到关联列表末尾

### 边界情况处理

| 情况 | 处理方式 |
| --- | --- |
| 摘要无「出场角色」板块 | 回退扫描原文文本，按角色名称全文匹配 |
| 摘要无「关联」板块 | 在文件末尾新增 `## 🔗 关联` 板块 |
| 章节对不齐（有原文无摘要） | 跳过处理，记录日志 |
| Windows 中文路径编码 | 使用 pathlib + UTF-8 确保正确处理 |
| 同一角色在本章出现多次 | 去重，只添加一次链接 |
| 角色名称歧义（如「张羽」也出现在「张羽母亲」） | 按精确匹配+长度优先规则处理 |


### 新增角色流程

脚本会输出一份「未匹配角色报告」，列出在原文/出场角色中出现、但角色目录中不存在的名称，人工确认后在对应子目录下创建标准角色档案文件（参考 `00-配角档案.base` 模板格式）。
